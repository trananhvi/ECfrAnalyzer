About eCFR Analyzer

Project Overview

The eCFR Analyzer is a comprehensive web application designed to analyze Federal Regulations from the Electronic Code of Federal Regulations (eCFR) API. This tool provides digestible and actionable insights to inform potential deregulation efforts across the U.S. government.

With over 200,000 pages of federal regulations across approximately 150 main agencies, this application helps identify patterns, complexity metrics, and areas where regulatory streamlining could provide the most benefit.

Key Features
Incremental Data Fetching

Automatically downloads eCFR data with smart incremental updates to minimize API calls and maintain current information.

Local Data Storage

Stores regulation data locally in JSON format for fast access and offline analysis.

Advanced Analytics

Provides comprehensive metrics including word counts, regulation counts, and our custom Regulatory Complexity Index.

Data Integrity

Generates checksums for each agency and overall dataset to track changes over time.

Custom Metric: Regulatory Complexity Index

Our signature metric, the Regulatory Complexity Index (RCI), is designed to identify agencies with the most complex regulatory frameworks that could benefit from deregulation efforts.

RCI Formula Components:
  • Verbosity Factor (40%): Average words per regulation normalized to a 1000-word baseline
  • Scope Factor (30%): Number of unique CFR titles covered by the agency
  • Structure Factor (30%): Complexity based on unique parts, chapters, and sections
Interpretation: Higher RCI values indicate agencies with more verbose, wide-reaching, and structurally complex regulations. These agencies represent prime candidates for regulatory streamlining initiatives.

USDS Take-Home Assessment

This application was developed as part of the United States Digital Service (USDS) Engineering Take-Home Assessment. The goal was to create a tool for analyzing federal regulations to support potential deregulation efforts.

Assessment Requirements Met:
  • Download current eCFR data
  • Store data server-side
  • Create data retrieval APIs
  • Provide analytical UI
  • Word count per agency
  • Agency checksums
  • Custom complexity metric
